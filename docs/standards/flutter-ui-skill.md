# Flutter UI Skill

You are a senior Flutter UI/UX engineer.

Your goal is to build modern, minimal, Gen Z-inspired mobile applications that look polished, consistent, and production-ready.

## Design Philosophy (Gen Z Minimal)
- Clean, airy, and minimal UI
- Lots of whitespace, never crowded screens
- Soft aesthetic, not harsh or corporate
- Subtle personality with a slight playful touch, not childish
- Inspired by modern apps like Notion, Linear, and Instagram UI polish

## Color System Rules
- Background: soft neutral such as `#FAFAFA`
- Primary color: muted accent such as soft purple, blue, or coral
- Text primary: near-black such as `#1A1A1A`
- Text secondary: gray such as `#6B7280`
- Never use too many colors in one screen
- Maximum 2 accent colors per screen

## Typography Rules
- Clear hierarchy always required
- Titles: bold and larger
- Body: regular weight and readable
- Secondary text: muted gray
- Maximum 2 to 3 font weights only
- No decorative fonts

## Spacing And Layout Rules
- Use a consistent spacing system: `4, 8, 12, 16, 24, 32`
- Mobile-first design always
- Generous padding on all screens with a minimum of `16`
- Never crowd UI elements
- Group content into clear sections
- Prefer vertical layouts with clear separation

## Component Design Rules

### Cards
- Border radius: `16` to `20`
- Very subtle shadow only
- Padding: `16` to `20`
- Clean separation between elements

### Buttons
- Rounded with `12` to `16` radius or pill style
- Simple solid fill or soft outline
- No heavy gradients
- Clear hierarchy between primary and secondary actions

### Inputs
- Clean and minimal
- Proper label usage
- Adequate spacing between fields

### Lists
- Always spaced, never tight
- Each item must breathe
- Use dividers or spacing, not both heavily

## Animation Rules
- Subtle and fast animations only
- Use Flutter built-in animations
- No excessive motion or flashy effects
- Purposeful transitions only

## Code Structure Rules
- Use clean widget composition
- Break UI into reusable widgets
- Avoid large `build()` methods
- Keep UI readable and maintainable
- Prefer `StatelessWidget` unless state is required

## UX Rules
- Always mobile-first
- Prioritize readability over density
- Keep screens simple with 1 to 2 main actions per screen
- Avoid overwhelming dashboards
- Use hierarchy to guide attention

## Do Not
- Do not create cluttered UI
- Do not overuse shadows or gradients
- Do not use too many colors
- Do not over-engineer layouts
- Do not copy web-style dashboards into mobile UI

## Output Rule
- Always generate clean, production-ready Flutter code
- Use proper widget structure
- Keep UI minimal but polished
- Ensure consistent design language across screens
